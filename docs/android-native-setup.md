# Native Android shell — build & run (handoff)

Pick-up guide for finishing the native Android app (`apps/android`) on a machine
with the Android toolchain. The app was **authored and its Rust/Kotlin contract
verified**, but never built/run on a device here (this machine has no Android
SDK/NDK). Everything it depends on — the `futo-notes-ffi` crate, the Kotlin
UniFFI bindings, the shared `editor.html`, the build script — is in place.

> The Android app is the sibling of `apps/ios`: native Compose chrome + a
> `WebView` hosting the **same** `editor.html`, with **all** note rules + CRUD +
> sync delegated to the **same** Rust `futo-notes-ffi` crate (Kotlin bindings).
> See `apps/android/README.md` for the layout.

## 1. Prerequisites

| Tool | Notes |
|---|---|
| **Android Studio** (or cmdline-tools) | Provides the SDK + `adb` + an emulator. Set `ANDROID_HOME` (e.g. `~/Library/Android/sdk`). |
| **Android NDK** | Install via Android Studio → SDK Manager → SDK Tools → "NDK (Side by side)". Then `export ANDROID_NDK_HOME="$ANDROID_HOME/ndk/<version>"`. |
| **JDK 17** | Required by the Android Gradle Plugin 8.5. |
| **cargo-ndk** | `cargo install cargo-ndk` |
| **Rust android targets** | `rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android i686-linux-android` |

`scripts/build-rust-android.sh` checks `ANDROID_NDK_HOME` + `cargo-ndk` up front
and fails with install instructions if either is missing.

## 2. One-shot build + run

With a device/emulator connected (`adb devices`):

```bash
just android-native        # == apps/android/run.sh
```

That script: builds `futo-notes-ffi` per-ABI → `jniLibs/<abi>/libfuto_notes_ffi.so`
+ Kotlin bindings, builds the shared `editor.html`, stages it into the app's
`assets/`, then `gradle :app:installDebug` and launches `com.futo.notes.native`.

## 3. Step by step (if the one-shot fails)

```bash
# a) Rust core (.so per ABI) + Kotlin UniFFI bindings
scripts/build-rust-android.sh
#    -> apps/android/app/src/main/jniLibs/<abi>/libfuto_notes_ffi.so
#    -> apps/android/app/src/main/java/uniffi/futo_notes_ffi/futo_notes_ffi.kt

# b) The shared editor bundle, staged into assets
pnpm exec vite build --config vite.editor.config.ts
mkdir -p apps/android/app/src/main/assets
cp apps/ios/Resources/editor.html apps/android/app/src/main/assets/editor.html

# c) Gradle build + install
cd apps/android
# No Gradle wrapper is committed. Either generate one (recommended):
gradle wrapper --gradle-version 8.9 && ./gradlew :app:installDebug
# ...or open apps/android in Android Studio and Run.
```

> **Gradle wrapper**: not committed (it carries a binary jar). Run
> `gradle wrapper` once (system Gradle ≥ 8.5) or just open the project in
> Android Studio, which provisions Gradle automatically.

## 4. Verify it works (parity with iOS)

Mirror the iOS smoke test:

1. App launches to the note list (seeded with "Welcome" + a "Specs" folder on a
   fresh install).
2. Tap a note → the CodeMirror editor renders (headings, bold/italic, a table,
   checkboxes, highlighted `#tags`).
3. `+` → New Note → type → back → the note shows in the list with its preview.
4. Create folders; move/rename notes.
5. Sync screen → connect to the E2EE server (emulator host = `http://10.0.2.2:<port>`).

## 5. Known gaps / things to check on device

- **Soft keyboard for an empty editor.** Android `WebView` (like iOS WKWebView)
  can refuse to raise the keyboard for an empty `contenteditable`. iOS solves
  this with a private-API swizzle (`EditorWebView.swift`); the Tauri Android app
  uses an "IME shield" (see `docs/learnings/ime-shield-workaround.md`). The
  native Compose `EditorWebView.kt` does **not** yet have an equivalent — if a
  brand-new note won't accept typing until you tap twice, port the IME approach.
- **`applicationId` = `com.futo.notes.native`** (distinct from the Tauri app's
  `com.futo.notes`) so both can coexist on one device.
- **`usesCleartextTraffic=true`** is enabled for local-server sync testing;
  tighten before any real distribution.
- The `futoBridge` contract is already cross-platform (`postToHost()` handles
  the Android `@JavascriptInterface` transport) — no editor changes needed.

## 6. Where things are

- App: `apps/android/` (`MainActivity.kt`, `NotesStore.kt`, `SyncManager.kt`,
  `ui/{NoteListScreen,NoteEditorScreen,SyncScreen,EditorWebView}.kt`).
- Rust facade: `crates/futo-notes-ffi` (same crate iOS uses).
- Build script: `scripts/build-rust-android.sh`.
- Generated `.so` + Kotlin bindings + staged `editor.html` are **gitignored**
  (regenerated by the build).
